## Overview
A web tool for managing and printing mech sheets for battletech.  

## Structure
This should be primarily a single-page app run all in the browser. Primarily we will use overlay dialogs for additional functionality like the print menu and what not. But the core experience will be driven in the main view entirely. I want a selection box for mech variants will all manner of filter capabilities. You should be able to filter on tonnage, BV, name, weapons, and so forth. I would also like the ability to mark that you own a particular mech mini and then view only those mech variants for which you have the mini. At its simplest, this can be a checkbox, but there should be a way of specifying how many of the mini you have in the case where someone has more than one. That can then be used later in creating lances and scenarios.  From the mech list, you should be able to drop mechs onto a lance/star editor. This should provide some overview information about the mech in question like tonnage and BV. You should also be able to change the piloting and gunnery skill on this view so that you can “right size” the BV. Provide a way to auto-balance the BV for a given target value by upping the piloting and gunnery skill using the standard matrix. This should then also provide a way of locking the skills for a given mech. We should have a display at the top of the lance that shows the total BV for the lance/star. There should be a way to save and reload lances that have been defined.  Any time you select a mech, it should show a preview of the mech in an ever present preview display. The mechs in the various drop downs should not display as simply lines but as cards with rich information about the mech.  

## Technology
Use whatever web-tech framework makes sense be it react, angular or some other framework. For persistence, we will start by just using the browser local storage. That said, it should be built such that it will be easy to add a better persistence layer likely using firebase with google login. Use node as the web-server, but it should be a really dumb web server, basically just serving up the static content. The source should be managed with git. Make regular commits.   

## Workflow
Feel free to create up to two additional agents. You can use git-subtree to work simultaneously and then merge things back into the main source.  

## Source Data
I have created a directory called data/ with some source data. Do not track any of this in the git repository. There is a directory called sheets. These are all of the mech variants you should support. Extract them from the sheets and convert them into a JSON structure which contains all of the pertinent data. I may add more sheets there in the future. You should also use these mech sheets in the PDF as a potential template for the mech sheets to print. I have also included a templates directory with examples of other ways to print mech sheets.  

## Print Format
This is the most important aspect of the app. I want to provide an easy way to print mech sheets in a number of different styles and formats. There is the standard format that you see in the provided PDFs. But I would also like to support something similar to the sheet-with-tables.pdf format that includes various hit tables. We will need to look at a number of different templates here and support selecting which style to use when printing. It should be possible to print a mech or a lance.  I would love to explore an abbreviated format that would make it possible to print an entire lance on just one sheet. This would likely require significantly redesigning the mech sheet format. For example, the internal critical hits table would need to be redesigned in a highly abbreviated form. This is something we can discuss later. But don’t let me forget about it once we’ve got the standard printing done.
